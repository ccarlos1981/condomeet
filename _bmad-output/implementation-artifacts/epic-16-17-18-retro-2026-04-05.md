## ✅ Sucessos & Pontos Fortes

### Entrega
1. **Módulo Assembleias completo em uma sessão** — 8 histórias entregues cobrindo o ciclo inteiro: criação → transmissão ao vivo → votação → ATA IA → faturamento.
2. **Reuso de padrões existentes** — O padrão Server Component → Client Component foi aplicado consistentemente em todas as novas rotas sem precisar reinventar.
3. **Degradação graciosa** — Vídeo via Agora.io funciona quando configurado, mas a UI mostra mensagem amigável em vez de quebrar se o App ID estiver ausente.
4. **Monetização integrada** — O faturamento foi desenhado como parte orgânica do fluxo (ATA IA → consumo registrado → dashboard super-admin), não como um "patch" posterior.

### Qualidade
5. **Zero erros de runtime** em produção após as correções (Building2 import fix).
6. **Separação de acesso segura** — Faturamento protegido no server-side por e-mail, não apenas por lógica de UI.
7. **Impressão A4** — ATA, Edital e Procuração com CSS `@media print` dedicado, pronto para uso jurídico/cartorial.

## ⚠️ Desafios & Áreas de Melhoria

### Encontrados e Resolvidos
1. **Modelo Gemini incorreto** — Inicialmente usamos `gemini-2.5-flash` que não existia; precisou correção para `gemini-2.5-flash-preview-04-17`. **Lição:** Sempre verificar disponibilidade de modelos antes de integrar.
2. **Import esquecido (Building2)** — Ícone usado no JSX sem estar no import. **Lição:** Após adições de componentes visuais, verificar imports automaticamente.
3. **Shadcn/ui indisponível** — Tentamos usar `Card`, `Select` do shadcn que não estavam instalados. **Lição:** Verificar dependências existentes antes de referenciá-las; usar Tailwind puro quando shadcn não está no projeto.

### Dívida Técnica Aceita
4. **Inline styles em barras de progresso** — 3 warnings de lint por `style={{ width: ${x}% }}`. Necessário para larguras dinâmicas calculadas. **Decisão:** Aceito como dívida técnica de baixo impacto.
5. **`aria-label` em selects de faturamento** — 2 warnings de acessibilidade. **Ação recomendada:** Adicionar `title` attribute nos selects na próxima iteração.

## 💡 Lições Aprendidas

| # | Lição | Impacto |
|---|-------|---------|
| 1 | **Verificar modelos de IA antes de integrar** — A API de modelos muda frequentemente. Sempre confirmar o nome exato do modelo disponível. | Alto — previne deploy quebrado |
| 2 | **Tailwind puro > shadcn quando shadcn não está instalado** — Não assumir que bibliotecas de componente estão disponíveis. | Médio — evita erros de build |
| 3 | **Faturamento deve nascer junto com o consumo** — Registrar custos no momento do uso (na Edge Function) é mais confiável do que tentar reconstituir depois. | Alto — padrão para futuras features monetizadas |
| 4 | **Fluxo de impressão requer CSS dedicado** — Documentos legais/cartoriais precisam de `@media print` desde o início, não retrofit. | Médio — garante usabilidade |
| 5 | **Filtros dinâmicos baseados em dados reais** — Mostrar apenas condomínios com consumo (não todos os cadastrados) reduz ruído. | Baixo — melhoria de UX |

## 🔍 Descobertas Significativas

### 1. Agora.io requer token server para produção
O modo "Testing Mode" (App ID only) funciona para desenvolvimento, mas em produção real com múltiplos condomínios simultâneos, será necessário implementar um token server para segurança dos canais de vídeo.

**Impacto:** Não bloqueia MVP, mas deve ser endereçado antes de escalar para 50+ condomínios.

### 2. Tabela `consumo_extras` como base de monetização
O padrão de registrar `tipo_servico`, `valor_base`, e `valor_cobrado` (3x) na mesma tabela será reutilizável para qualquer futuro serviço cobrado (transcrição, OCR avançado, etc.).

**Impacto:** Positivo — padrão escalável definido.

### 3. O módulo de Assembleias é o maior do sistema
Com ~21.500 LOC e 34 arquivos, o módulo de Assembleias é significativamente maior que qualquer outro módulo individual. Merece atenção especial em testes e manutenção.
